1. 打开对应组别的评分标准文档
2. 逐个TODO对照考生代码
3. 根据评分标准表格给分
4. 汇总总分

5. 人工抽查10%-20%进行质量控制

## 🔍 评分要点提示

### 完全正确 (100%分数)
- 代码与标准答案一致
- 使用等效方法且功能正确

### 部分正确 (70%-90%分数)
- 逻辑正确但参数不完全匹配
- 拼写错误但意图明确
- 使用替代方案且基本可用

### 基本正确 (40%-60%分数)
- 思路正确但实现有误
- 函数名对但参数错误

### 错误 (0分)
- 完全错误或空白
- 语法错误无法运行

---

## ⚠️ 特别注意事项

### 1. 容错原则
- 字符串格式化方法不限（f-string/.format()/拼接）
- 单引号双引号等效
- 功能等效的替代方法给满分

### 2. 酌情扣分
- 参数在合理范围变化：扣10%-20%
- 函数名对参数错：扣20%-40%
- 架构整体正确但细节不同：扣20%-30%

### 3. 必须人工复查的情况
- 总分<40分（可能误判）
- 总分>95分（可能过于宽松）
- 使用了创新性解法
- 评分理由不充分

## 📝 常见问题FAQ

### Q1: 考生使用了标准答案中没有的方法怎么办？
**A**: 如果功能等效且能正常运行，应给满分。例如使用`flatten()`替代`ravel()`。

### Q2: 模型通道数配置不同但整体架构合理怎么评分？
**A**: Task2的模型搭建(8分)，如果整体合理但通道数不同，给6-7分。

### Q3: 职工组的双向LSTM输出维度是关键点吗？
**A**: 是的！必须是`128*2`或`256`，这是职工组的重要考点(3分)。

### Q4: GUI部分的按钮文本写错了扣多少分？
**A**: 按钮文本(1分)，如果功能绑定正确但文本错误，扣0.5-1分。

### Q5: 大模型评分可靠吗？
**A**: 需要配合人工抽查。建议：
- 10%-20%抽查率
- 边界分数必须复查
- 多模型交叉验证

## 📈 质量保证

### 评分一致性检查
1. 同一份试卷由2个评分员评分，误差应<5分
2. 大模型评分与人工评分误差应<8分
3. 相同错误在不同试卷中扣分应一致

### 申诉处理
对于考生申诉：
1. 重新评阅该TODO
2. 查阅是否属于"可接受变体"
3. 如确有错误，调整分数并记录
4. 更新评分标准文档
